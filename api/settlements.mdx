## Create Settlement

<api method="POST" path="/api/settlements" auth="bearer" />

Create a new settlement for a group. This calculates the minimum number of transactions needed to settle all debts and marks the group as settled.

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Request Body

<ParamField body="group" type="string" required>
  Group ID to create settlement for
</ParamField>

### Response

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

<ResponseField name="settlement" type="object">
  Settlement object containing transaction details
  <Expandable title="Settlement object structure">
    <ResponseField name="_id" type="string">
      Settlement's unique identifier
    </ResponseField>
    <ResponseField name="group" type="string">
      Group ID
    </ResponseField>
    <ResponseField name="type" type="string">
      Settlement type (always "group" for this endpoint)
    </ResponseField>
    <ResponseField name="status" type="string">
      Settlement status: `pending`, `completed`, or `cancelled`
    </ResponseField>
    <ResponseField name="createdBy" type="string">
      User ID of the person who created the settlement
    </ResponseField>
    <ResponseField name="summary" type="array">
      Array of transaction objects showing who should pay whom
      <Expandable title="Transaction object">
        <ResponseField name="from" type="object">
          User object (debtor) with _id and name
        </ResponseField>
        <ResponseField name="to" type="object">
          User object (creditor) with _id and name
        </ResponseField>
        <ResponseField name="amount" type="number">
          Amount to be transferred
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash cURL
curl -X POST https://api.billbuddy.com/api/settlements \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
```

<api method="PUT" path="/api/settlements/:id/status" auth="bearer" />

<ParamField header="Authorization" type="string" required>

## Understanding Settlement Calculations

<Info>
  **How the settlement algorithm works:**
  
  1. **Calculate Balances**: For each member, calculate total amount paid minus total amount owed
  2. **Identify Creditors and Debtors**: Positive balances are creditors (owed money), negative balances are debtors (owe money)
  3. **Minimize Transactions**: Use a greedy algorithm to match the largest debtor with the largest creditor
  4. **Generate Summary**: Create a list of transactions that settles all debts with minimum transfers
</Info>

### Example Scenario

Imagine a group of 3 people with these expenses:
- John paid $150 for groceries (split 3 ways)
- Alice paid $90 for gas (split 3 ways)
- Bob paid $60 for supplies (split 3 ways)

**Balances:**
- John: +$150 - $100 = +$50 (owed $50)
- Alice: +$90 - $100 = -$10 (owes $10)
- Bob: +$60 - $100 = -$40 (owes $40)

**Settlement transactions:**
- Bob pays John $40
- Alice pays John $10

**Result:** All debts settled with just 2 transactions instead of potentially 6.
